Repositório de extensões para QGIS

Desde Agosto 2014 o Grupo utilizadores QGIS PT disponibiliza um repositório de extensões (plugins) para QGIS onde irão ser publicadas ferramentas criadas pelo grupo, para o uso no contexto das tarefas/dados específicas/os de Portugal.
O URL do repositório é o seguinte:

http://www.qgis.pt/plugins/plugins.xml


Para acrescentar o repositório no QGIS siga as seguintes opções:

Plugins -> Manage and Install Plugins -> Settings -> Plugins Repositories -> Add...

e preencha os detalhes assim como mostrado na seguinte imagem:

Extensões disponíveis

Tranformações de Datum para caixa de ferramentas Processing

Esta extensão foi criada para tornar o mais simples possível a tarefa da transformação de dados vectoriais e raster em sistemas de coordenadas obsoletos (Datum 73, Datum Lisboa e ED50) para o novo sistema de coordenadas oficial para Portugal continental (ETRS89-PTTM06), utilizando o metodo das grelhas de tranformação de Datum (NTv2).

As grelha suportadas são as disponibilizadas pelo Prof. José Alberto Gonçalves (UP) e pela Direção-Geral do Território (DGT):

http://www.fc.up.pt/pessoas/jagoncal/coordenadas/

http://www.dgterritorio.pt/cartografia_e_geodesia/geodesia/transformacao_de_coordenadas/grelhas_em_ntv2/


As grelhas são disponibilizadas/instaladas em conjunto com a extensão e portanto não é pedida nenhuma configuração manual por parte do utilizador. A extensão foi criada para ser o mais simples possível para o utilizador: as ferramentas são organizadas por tipo de dados (vectores ou raster) e por sistemas de coordenadas dos dados de origem (Datum 73, Datum Lisboa ou ED50):

Notas:

  • O desenvolvimento do plugin foi suportado pelo Grupo utilizadores QGIS PT e realizado pelo mesmo grupo, pela Faunalia e Alexander Bruy
  • Como qualquer outra ferramenta da toolbox Processing de QGIS, tambem estas ferramentas podem funcionar em modalidade batch, tornando muito simples a tarefa da tranformação de coordeandas de conjuntos de dados grandes
  • Repositório do código fonte: https://github.com/qgispt/processing_pttransform
  • Bug tracker: https://github.com/qgispt/processing_pttransform/issues
  • A extensão requer QGIS versão 2.4 ou master (a próxima versão 2.6) para funcionar.
  • É necessário verificar que na pasta .qgis2/python/plugins não seja presente uma pasta "processing". Se for presente é necessário apagar a mesma antes de instalar a extensão.

Problemas conhecidos:

  • Nas ferramentas de transformação de raster há um bug conhecido no QGIS 2.4, mas já resolvido no QGIS master: se a opção "open output file after running algorithm" for activa então o raster será carregado com o sistema de coordenadas errado (o dos dados de input). Se o raster for carregado manualmente no QGIS o problema não se apresenta.
  • Nalguns casos no QGIS 2.4 poderá aparecer um aviso (não um erro) sem uma aparente razão real: Layers do not all use the same CRS. This can cause unexpected results. Do you want to continue?. O aviso pode-se ignorar e de qualquer forma o problema já foi resolvido no QGIS master.
  • Os paths dos ficheiros de input/output não devem conter caracteres especiais (espaços, acentos, tildes, etc.).
  • As ferramentas de transformação de vectores funcionam de momento utilizando como input só o formato Shapefile.
